リペイント問題：TradingViewインジケーターの多くが欺く方法

7分で読めます Signal Pilotチーム
共有：

完璧なバックテストのそのインジケーター？外科的な精度ですべての天井と底を捉えるもの？

あなたに嘘をついています。

たぶんではなく。時々ではなく。もし過去のシグナルが真実であるには良すぎるように見えるなら、そのインジケーターはほぼ確実にリペイントしています - そしてライブで取引した瞬間に高い授業料を払うことになります。

リペイントが本当に意味すること

リペイントとは、インジケーターが過去のシグナルを後から変更することです。正確な底で現れる買いの矢印？ローソク足が形成されているときにはそこにありませんでした。インジケーターは何が起こるか見てから、遡ってシグナルを描いたのです。

これは昨日の天気を「予測」する気象予報士のようなものです。結果をすでに知っているとき、正しいのは簡単です。

実際にはこのように見えます：

ローソク足の形成中： シグナルは現れません。価格は上下しています。あなたは見て、待っています。

ローソク足が閉じた後（価格が上がった）： 突然そのローソク足に買いシグナルが現れます。インジケーターはずっと「知っていた」のです。

ローソク足が閉じた後（価格が下がった）： シグナルなし。インジケーターは黙っています、まるで買うことを考えたこともなかったかのように。

インジケーターは何も予測していません。賢く見せるために歴史を書き換えているのです。

なぜリペイントするインジケーターが存在するのか

2つの理由：無知と欺瞞。

無知： 一部の開発者は本当に問題を理解していません。インジケーターを作り、素晴らしい過去の結果を見て、それが機能すると思い込みます。誰も騙そうとしているのではなく - 単に知らないだけです。

欺瞞： 他の人は自分が何をしているか正確に知っています。リペイントするインジケーターはマーケティングが簡単です。バックテストは信じられないほど良く見えます。スクリーンショットは完璧なエントリーを示します。まだライブで取引していない人々から証言が集まります。

ユーザーが気づく頃には、開発者は次の「革命的な」システムに移っています。

リペイントの3つのタイプ

すべてのリペイントが同じではありません。タイプを理解することで問題を特定できます：

タイプ1：シグナルの移動

シグナルが1つのローソク足に現れ、データが追加されると別のローソク足にジャンプします。昨日見た「完璧な」エントリーは今や3本前のローソク足に移動しています。歴史が書き換えられました。

タイプ2：シグナルの消失

ライブ取引中に現れたシグナルがチャートから消えます。インジケーターが買いシグナルを出し、トレードを取り、失敗しました - そして今やシグナルはもうそこにありません。あなたのトレードをレビューする人にそれを説明する幸運を。

タイプ3：値の再計算

インジケーターの値自体が遡って変わります。買われ過ぎを示していたオシレーターが今は中立を示しています。昨日あったサポートレベルが移動しています。全履歴記録が信頼できません。

任意のインジケーターでリペイントをテストする方法

マーケティング、スクリーンショット、バックテストはすべて誤解を招く可能性があります。自分でテストしてください。

スクリーンショット方法：

  1. ライブチャートにインジケーターを追加
  2. シグナルがある最後の10-20本のローソク足のスクリーンショットを撮る
  3. 10-20本の新しいローソク足が形成されるまで待つ
  4. 同じ履歴エリアの別のスクリーンショットを撮る
  5. 2つの画像を比較

何かが動いた、消えた、または変わった場合 - リペイントしています。以上。

ライブ観察方法：

  1. 低い時間枠でチャートを開く（5分がよく機能します）
  2. 1時間リアルタイムでインジケーターを観察
  3. ローソク足形成中に現れるすべてのシグナルをメモ
  4. 各ローソク足が閉じた後、シグナルがまだあるか確認
  5. セッション後、戻ってすべてのシグナルが記録したものと一致するか確認

これには時間がかかります。価値があります。1時間のテストで何ヶ月もの挫折と何千もの損失を節約できます。

質問方法：

開発者に直接聞いてください：「このインジケーターはリペイントしますか？」

回答を注意深く観察：

  • 「いいえ」→ 良いスタートですが、とにかく確認
  • 「現在のローソク足でのみ」→ それでもリペイントです。現在のローソク足だけが重要です。
  • 「精度のために再計算します」→ リペイントします。それが再計算の意味です。
  • 回答なし/曖昧な回答 → リペイントします。隠しています。

「現在のローソク足でのリペイント」がそれでもあなたを破壊する理由

一部の開発者は、インジケーターが「現在のローソク足でのみリペイントする」と主張し、それが許容できるかのように言います。許容できません。

理由はこうです：

チャートを見ています。買いシグナルが現れます。トレードに入ります。ローソク足が閉じ - シグナルが消えます。インジケーターは、終値を見た後、実際には買いではなかったと決定しました。

しかしあなたはすでにトレード中です。

これは小さな不便ではありません。ライブ取引では完全に役に立たないインジケーターです。重要なシグナルは、あなたが行動できるものだけです。それらのシグナルが消える可能性があるなら、どれも信頼できません。

非リペイントが実際にどう見えるか

正しく構築されたインジケーターはこのように機能します：

  1. シグナル条件はローソク足の終値で評価される
  2. シグナルが一度現れたら、永遠に残る
  3. 履歴シグナルはライブで現れたものと正確に一致
  4. 再計算なし、移動なし、消失なし

シグナルは現れるか現れないか。一度そこにあれば、永続的です。履歴で見るものは、ライブ取引で見たであろうものと正確に同じです。

これは最低基準です。機能ではなく - 要件です。

これを無視するコスト

知らずにリペイントするインジケーターを使用するトレーダーは以下を経験します：

偽りの自信： バックテストは利益があるように見えるので、サイズを上げます。その後現実が襲います。

幻のセットアップ： もうチャートに存在しないシグナルを見た記憶があります。疑念が忍び寄ります。「想像したのか？」

システムのジャンプ： インジケーターが「機能しなくなった」（元々機能していなかった）ので、次へ移ります。サイクルが繰り返されます。

口座へのダメージ： 実際には存在しなかったシグナルで本物のお金を失いました。

心理的ダメージは金銭的ダメージより悪いかもしれません。リペイントするインジケーターで痛い目に遭ったトレーダーは、正当なものでさえ、どんなツールも再び信頼するのに苦労することがよくあります。

結論

実際の取引に任意のインジケーターを使用する前に、リペイントをテストしてください。任意のインジケーターを購入する前に、リペイントしない証拠を要求してください。任意のバックテストを信頼する前に、シグナルがリアルタイムで存在したことを確認してください。

これはオプションではありません。偏執的ではありません。お金をリスクにさらすツールの基本要件です。

テストに費やす10分で、何ヶ月もの損失と取引への完全な信頼の危機を節約できるかもしれません。

実際に機能するもの

非リペイントはベースラインであるべきで、例外ではありません。しかしそれは出発点に過ぎません。

最高のインジケーターシステムは複数の分析次元を組み合わせます - サイクル検出、ボリューム分析、モメンタム投票、キーレベル - そのどれもリペイントしません。7つの独立したシステムがシグナルに同意し、どれも遡って投票を変更できないとき、あなたは本当に信頼できる何かを見ています。

それが私たちがSignal Pilotを構築したものです：すべてのインジケーターがローソク足の終値で確定し、履歴データがライブで現れたものと正確に一致し、コンフルエンスが複数の非リペイントシステムで同時に測定されるスイートです。

私たちの言葉を信じる必要はありません。上記で説明した正確な方法で自分でテストしてください。何かがリペイントしたら、使用しないでください。その基準は私たちにも適用されます。
